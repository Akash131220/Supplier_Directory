----------------------------------------
🎯 OBJECTIVE
----------------------------------------
When a user searches for a supplier and NO result is found:

→ Automatically store that search input into a separate Supabase table  
→ So admin can later review and add missing supplier data

----------------------------------------
🗄️ DATABASE DESIGN
----------------------------------------

Create a new table:

Table Name: missing_suppliers

Columns:
- id (uuid, primary key, auto-generated)
- supplier_code (text, nullable)
- supplier_name (text, nullable)
- search_query (text, required)
- created_at (timestamp, default now())
- status (text, default 'pending')

----------------------------------------
⚙️ LOGIC FLOW
----------------------------------------

1. User enters search input:
   Example:
   - "SUP0120"
   - "Anil Traders"

2. System searches in suppliers table

   → Show supplier data

4. IF NO RESULT FOUND:
   → Show "No supplier found"

   AND ALSO:

   👉 Save entry in missing_suppliers table:
   - If input matches supplier code pattern → store in supplier_code
   - Else → store in supplier_name

   - Always store full search in search_query

----------------------------------------
🧠 SMART DETECTION LOGIC
----------------------------------------

If input contains:
- "SUP" + numbers → treat as supplier_code
Else:
- treat as supplier_name

----------------------------------------
🔁 DUPLICATE HANDLING
----------------------------------------

Before inserting:
→ Check if same search_query already exists

If exists:
→ DO NOT insert again

🎨 UI/UX

⚡ PERFORMANCE

- Insert only once per unique query

🔐 SECURITY

- Only admin can view missing_suppliers table
